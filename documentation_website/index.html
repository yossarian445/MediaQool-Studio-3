Motivation

In Germany alone, there are 21 million smartphone owners who record videos with their phones regularly. When a lot people are filming simultaneously during social events like concerts or sport events, those videos tend to feature very similar content. If the videos were collected in a single database like MediaQ, this would facilitate a broad range of applications for crowdsourcing. The different camera angles could be used to generate a new viewing experience for social events.

Our goal:

With MediaQool Studio 3 we want to provide an easy to use tool for amateur film editors, which enables them to select video footage of certain events in a way that meets their individual requirements. The videos are automatically synchronized to a timeline, so the user can directly import the videos to the video editing application of his choice.

Concept and Requirements

In the creation process of MediaQool Studio 3 we decided to put the main focus of the project on the user-interface and usability. The following requirements reflect this decision and put emphasis on the front-end architecture and design.

  • The user should be able to efficiently select from all available videos in the database
  • The user should get a fast overview over video quality and content
  • The user should be provided with a downloadable, platform-independent “video-files + project-file” bundle
  • The user interface should be appealing and easy to use with focus on simplicity

Execution

In order to meet the defined requirements, we decided to split the video selection process into two different views. The first view, featuring a map and a timeslider, is used for a rough selection of video items, which means the user can refine the selection by entering a time frame and choosing an area in which the desired videos were shot. The second view displays all previously selected videos in a timeline, where they are positioned according to their duration and start time. Furthermore the second display includes a thumbnail preview for each video item, so that the user can get a quick view of the content and quality of the video. The user can now further refine the search and select the videos he wants to use in the video editing process.

Here you can see how this design fits the requirements:

  • The user should be able to efficiently select from all available videos in the database
    • → Create UI where the user can select a time frame and area
  • The user should get a fast overview over video quality and content
    • → Create a view where the user can quickly preview (and if required deselect) videos
  • The user should be provided with a downloadable, platform-independent “video-files + project-file” bundle
    • → Look for platform-independent video-project formats, find a way to parse such files, implement a download-functionality

Solutions

Fast Video Forwarding

  • We encountered some performance problems while playing through the actual video files.
  • Solution
    • Generate spritesheets with video-screenshots
    • For every video one Spritesheet with Screenshot every 4 seconds
    • Move ‘viewing window’ over spritesheet

Parsing The Video Project

  • Solution:
    Final Cut Pro XML Interchange Format
    “was created so that every detail of your project file can be transferred to a system that doesn’t recognize native Final Cut Pro projects”

Application

Selection View

  • Provides a user-friendly way to select a time frame and area
  • Used technology:
    Javascript (incl. AJAX, jQuery), Google Maps API, various UI libraries/widgets (jQuery UI, Ion, ...)
  • Biggest challenge: Performance (lots of server requests)
    → Solution: Caching

Timeline View

File-Import with Adobe Premiere Pro

  • Open Adobe Premiere Pro
  • Import your XML-File with File → Import
  • In the Project-Window right click on the project folder → Link Media
  • Select one of the videos, the rest should be linked automatically ...

Server

  • Executes queries to MediaQ-Server (same-origin-policy)
  • Generates spritesheets and meta-files
  • Serves (mostly via AJAX):
    • HTML and related files
    • Spritesheets and Meta-Data (JSON)
    • MediaQ-Data
  • Frameworks
    • NodeJS + Express
    • FFmpeg + ImageMagick (for generating thumbs and spritesheets)
    • Various Node Packages (some modified by us)
